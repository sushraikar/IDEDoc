Troubleshooting Guide

This guide provides solutions to common issues you might encounter when working with the UAE Real-Estate AI Platform in the Windsurf Editor environment. If you're experiencing problems, follow the troubleshooting steps below to resolve them.

Common Issues and Solutions

This section covers the most common issues encountered when working with the UAE Real-Estate AI Platform in the Windsurf Editor environment, along with their solutions.

MCP Server Connection Issues

Symptoms:

  • AI Flows fail with "Unable to connect to MCP server" error
  • Tools are not available in the Windsurf Editor
  • Connection timeout errors when executing flows

Possible Causes:

  • MCP server is not running
  • Incorrect server URL configuration
  • Network connectivity issues
  • Port conflicts

Solutions:

  1. Verify MCP server is running: 
    ps aux | grep mcp-server

    If not running, start it with:

    cd /path/to/mcp-server
npm run start
  2. Check server URL configuration:

    Ensure the MCP server URL in your Windsurf Editor settings matches the actual server address. The default is:

    http://localhost:3000
  3. Test server connectivity: 
    curl http://localhost:3000/health

    You should receive a response like:

    {"status":"ok"}
  4. Check for port conflicts: 
    lsof -i :3000

    If another process is using port 3000, either stop that process or change the MCP server port in the .env file:

    PORT=3001
  5. Check environment variables:

    Ensure all required environment variables are set in the .env file:

    PORT=3000
VECTOR_DB_CONNECTION=postgresql://username:password@localhost:5432/vectordb
BAYUT_API_KEY=your_bayut_api_key
PROPERTY_FINDER_API_KEY=your_property_finder_api_key
DLD_API_KEY=your_dld_api_key
OPENAI_API_KEY=your_openai_api_key
